## 1. Клонирование репозитория
bash
```
git clone https://github.com/NoDreams2/android_weather.git
```
```
cd android_weather
```

## 2. Настройка local.properties
Создайте файл local.properties в корне проекта и укажите путь к Android SDK:
```
echo "sdk.dir=C\:\\Users\\%USERNAME%\\AppData\\Local\\Android\\Sdk" > local.properties
```

## 3. Сборка APK
```
./gradlew clean
```
```
./gradlew assembleDebug
```

## 4. Где найти APK
app/build/outputs/apk/debug/app-debug.apk

# О проекте
При запуске приложения открывается главная страница с погодой. В верхней части страницы есть окно поиска, в котором можно ввести название города. После ввода и отправки запроса на странице отображается текущая погода в выбранном городе.

Справа от поиска находится кнопка с геометкой. При нажатии на нее открывается карта с достопримечательностями Москвы.

При нажатии на любую метку открывается диалоговое окно с описанием данной достопримечательности.

Внизу главной страницы расположена кнопка «Воспоминания». При нажатии на нее открывается страница заметок. На этой странице есть плавающая кнопка для создания новой записи. При нажатии на кнопку «Сохранить» запись сохраняется и отображается в списке. Каждая запись имеет кнопку «Удалить» для ее удаления.

# Монетизация и публикация
Планируется публикация приложения в Google Play Store. Модель монетизации — бесплатное приложение с рекламой:

В приложении будут отображаться ненавязчивые рекламные баннеры (например, внизу главного экрана) и вознаграждаемые видеоролики (например, за разблокировку дополнительных функций).

В качестве рекламной платформы планируется использовать Google AdMob.

Дополнительная монетизация возможна за счет премиум-подписки, отключающей рекламу и открывающей расширенные возможности (например, прогноз погоды на неделю или возможность добавлять неограниченное количество заметок).

Приложение будет опубликовано под аккаунтом разработчика с заполненной карточкой в Google Play, включающей описание, скриншоты, политику конфиденциальности и возрастные ограничения.

# Безопасность приложения
Для обеспечения безопасности данных пользователя и корректной работы приложения реализованы следующие меры:

Хранение данных: заметки пользователя сохраняются в локальной базе данных SQLite, данные не передаются на внешние серверы и остаются только на устройстве.

Сетевые запросы: все запросы к API (погода) выполняются по защищенному протоколу HTTPS, что исключает перехват и подмену данных.

API-ключи: ключи доступа к внешним сервисам (OpenWeatherMap, Яндекс.Карты) хранятся в коде приложения и не передаются в открытом виде через сеть.

Минимальные разрешения: приложение запрашивает только те разрешения, которые необходимы для работы (доступ к интернету, геолокация для карты).

Отсутствие сбора персональных данных: приложение не собирает и не передает личную информацию о пользователе.
